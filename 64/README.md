# 变分蒙特卡洛（VMC）计算氢原子基态能量
## （自适应步长 + 随机重配置 + 虚拟时间演化）

## 项目简介

本项目使用变分蒙特卡洛方法计算氢原子的基态能量，采用简单试波函数 Ψ(r) = e^(-αr)，其中 α 是变分参数。

**主要特性：**
1. ✅ **自适应步长调整**：自动将Metropolis接受率维持在40%-60%的最优范围
2. ✅ **随机重配置**：使用Stochastic Reconfiguration方法计算能量梯度
3. ✅ **虚拟时间演化**：通过Propagator自动优化波函数参数，无需手动网格扫描

## 新增功能亮点

- **无需手动选择初始参数范围**：从任意初始α开始自动收敛到最优值
- **实时梯度计算**：每步蒙特卡洛采样同时计算能量对参数的梯度
- **自适应学习率**：虚拟时间演化的步长可根据收敛情况调整
- **收敛监控**：实时追踪α、能量、梯度的变化过程

## 理论背景

### 变分原理
对于任何试探波函数 Ψ，其能量期望值满足：
⟨E⟩ = ⟨Ψ|H|Ψ⟩ / ⟨Ψ|Ψ⟩ ≥ E₀

其中 E₀ 是基态精确能量。

### 氢原子哈密顿量
H = -½∇² - 1/r （原子单位）

### 局域能量
对于试波函数 Ψ = e^(-αr)，局域能量为：
E_L(r) = (HΨ)/Ψ = -½α² + (α-1)/r

## 代码结构

### VMC_Hydrogen 类

#### 初始化参数
- `alpha`: 变分参数
- `n_walkers`: 行走者数量（默认1000）
- `n_steps`: 蒙特卡洛步数（默认1000）
- `step_size`: Metropolis步长（默认1.0）

#### 主要方法

1. **wave_function(r)**: 计算试波函数值
   ```python
   def wave_function(self, r):
       return np.exp(-self.alpha * r)
   ```

2. **local_energy(r)**: 计算局域能量
   ```python
   def local_energy(self, r):
       return -0.5 * self.alpha**2 + (self.alpha - 1.0) / r
   ```

3. **metropolis_step(positions)**: 执行Metropolis采样
   - 生成新位置
   - 计算接受概率
   - 根据概率接受或拒绝

4. **run(thermalization)**: 运行VMC计算
   - thermalization: 热化步数（默认200）
   - 返回能量样本数组和平均接受率

### optimize_alpha 函数

在给定范围内优化变分参数 α，寻找使能量最小的最优值。

## 可调参数说明

- **n_walkers**: 增加行走者数量可以提高统计精度，但增加计算时间
- **n_steps**: 增加蒙特卡洛步数可以减小统计误差
- **step_size**: 调整步长使接受率在40%-60%之间最优
- **thermalization**: 热化步数应足够让系统达到平衡分布

## 数学推导

对于试波函数 Ψ = e^(-αr)：

1. 归一化波函数：
   Ψ(r) = (α³/π)^(1/2) e^(-αr)

2. 动能期望值：
   ⟨T⟩ = ½α²

3. 势能期望值：
   ⟨V⟩ = -α

4. 总能期望值：
   ⟨E⟩ = ½α² - α

对 α 求导并令导数为零，得到最小值：
d⟨E⟩/dα = α - 1 = 0 ⇒ α = 1

此时最小能量为：
⟨E⟩_min = -½ Hartree

## 自适应步长详解

### 问题背景
固定步长的缺陷：
- **步长过大**：接受率 < 10%，大部分提议被拒绝，采样效率极低
- **步长过小**：接受率 > 90%，随机行走几乎原地踏步，探索空间极慢

### 自适应算法原理

**目标：** 将接受率维持在 40%-60% 的最优范围

**调整策略：**
```python
def adjust_step_size(self, accept_rate):
    if accept_rate > self.target_accept_rate:
        self.step_size *= self.adjust_factor  # 接受率太高，增大步长
    else:
        self.step_size /= self.adjust_factor  # 接受率太低，减小步长
```

**调整频率：**
- 每 `adjust_interval` 步计算一次平均接受率
- 仅在**热化阶段**进行调整，采样阶段保持步长固定

**典型参数：**
- `target_accept_rate = 0.5`（目标50%接受率）
- `adjust_interval = 10`（每10步调整一次）
- `adjust_factor = 1.05`（每次调整±5%）

### 自适应效果
- **自动寻优**：无需手动调试步长
- **鲁棒性强**：对不同α值自动选择合适步长
- **采样高效**：接受率稳定在最优范围，马尔可夫链混合更快

## 注意事项

1. Metropolis采样的接受率应保持在合理范围（40%-60%）
2. 热化步数要足够，确保系统脱离初始分布并完成步长调整
3. 统计误差随 √N 减小，N为独立样本数
4. 为获得更高精度，需要增加计算资源
5. 自适应仅在热化阶段进行，采样阶段步长保持固定
