# 信用评分卡使用说明

## 功能概述

本项目实现了一个完整的基于逻辑回归的信用评分卡系统，支持：

### 核心功能
- 接收申请人特征数据（收入、负债比、历史逾期等）
- 计算违约概率
- 输出信用评分
- 给出风险等级评估

### 高级功能
- **单调分箱功能**：确保分箱后的违约率符合业务逻辑的单调性
- **拒绝推论（Reject Inference）**：对被拒客户推断违约标签，纠正样本偏差

## 使用方法

### 1. 快速开始（基础版）

```python
import pandas as pd
from credit_scorecard import CreditScorecard, create_sample_data

# 创建示例数据
df = create_sample_data(2000)
X = df[['income', 'debt_ratio', 'past_due', 'credit_lines', 'age']]
y = df['target']

# 训练评分卡模型（默认启用单调分箱）
scorecard = CreditScorecard(target_score=600, target_odds=60, pdo=50)

# 可选：手动设置单调性趋势
scorecard.set_monotonic_trend('age', 'decreasing')      # 年龄越大，违约率越低
scorecard.set_monotonic_trend('income', 'decreasing')   # 收入越高，违约率越低
scorecard.set_monotonic_trend('debt_ratio', 'increasing')  # 负债比越高，违约率越高
scorecard.set_monotonic_trend('past_due', 'increasing')    # 逾期次数越多，违约率越高

scorecard.fit(X, y, binning_method='equal_freq', n_bins=5, enforce_monotonicity=True)

# 预测新申请人
new_applicant = pd.DataFrame([{
    'income': 80000,
    'debt_ratio': 0.2,
    'past_due': 0,
    'credit_lines': 5,
    'age': 35
}])

# 获取预测结果
results = scorecard.predict(new_applicant)
print(results)
```

### 2. 快速开始（含拒绝推论）

```python
import pandas as pd
from credit_scorecard import CreditScorecard, create_approved_rejected_data

# 创建审批数据（含批准和被拒客户）
approved, rejected = create_approved_rejected_data(total_samples=2000, reject_rate=0.3)

features = ['income', 'debt_ratio', 'past_due', 'credit_lines', 'age']
X_approved = approved[features]
y_approved = approved['target']
X_rejected = rejected[features]

# 使用拒绝推论训练模型
scorecard = CreditScorecard(target_score=600, target_odds=60, pdo=50)
scorecard.set_monotonic_trend('age', 'decreasing')
scorecard.set_monotonic_trend('income', 'decreasing')
scorecard.set_monotonic_trend('debt_ratio', 'increasing')
scorecard.set_monotonic_trend('past_due', 'increasing')

# 方法1：硬截断法
scorecard.fit_with_reject_inference(
    X_approved, y_approved, X_rejected,
    method='hard_cutoff',
    cutoff=0.3  # 可选：自定义截断概率
)

# 方法2：模糊法（推荐）
scorecard.fit_with_reject_inference(
    X_approved, y_approved, X_rejected,
    method='fuzzy',
    sample_weight=1.0  # 推断样本权重
)

# 方法3：扩充法
scorecard.fit_with_reject_inference(
    X_approved, y_approved, X_rejected,
    method='augmentation',
    weight_factor=1.5  # 样本权重系数
)
```

### 3. 主要类和方法

#### CreditScorecard 类

**初始化参数：**
- `target_score`: 目标评分（默认600）
- `target_odds`: 目标赔率（默认60，表示60:1的好客户:坏客户比例）
- `pdo`: Points to Double Odds（默认50，表示评分每增加50分，赔率翻倍）

**主要方法：**

1. `set_monotonic_trend(feature, trend)`
   - 设置特征的单调性趋势
   - `feature`: 特征名称
   - `trend`: 'increasing'（违约率递增）或 'decreasing'（违约率递减）

2. `fit(X, y, binning_method='equal_freq', n_bins=5, enforce_monotonicity=True)`
   - 训练评分卡模型（不使用拒绝推论）
   - `X`: 特征数据（DataFrame）
   - `y`: 目标变量（0=好客户，1=违约）
   - `binning_method`: 分箱方法，'equal_freq'（等频）或 'equal_width'（等宽）
   - `n_bins`: 分箱数量
   - `enforce_monotonicity`: 是否强制单调性（默认True）

3. `fit_with_reject_inference(X_approved, y_approved, X_rejected, method='fuzzy', **kwargs)`
   - 使用拒绝推论训练评分卡模型
   - `X_approved`: 批准客户的特征
   - `y_approved`: 批准客户的表现标签
   - `X_rejected`: 被拒客户的特征
   - `method`: 拒绝推论方法，可选 'hard_cutoff', 'fuzzy', 'augmentation'

4. `predict(X)`
   - 返回完整预测结果
   - 包含：违约概率、信用评分、风险等级

5. `predict_proba(X)`
   - 仅返回违约概率

6. `calculate_score(X)`
   - 仅返回信用评分

7. `get_feature_scores(X)`
   - 返回各特征对评分的详细贡献

8. `print_binning_monotonicity_report(df, target)`
   - 打印分箱单调性验证报告

### 4. 单调分箱原理

**问题：**
普通分箱（等频/等宽）只考虑数值分布，可能导致分箱后的违约率出现"震荡"，例如年龄分箱后违约率呈现：25岁→20%，35岁→15%，45岁→18%（出现反弹）。

**解决方案：**
单调分箱算法通过以下步骤确保单调性：
1. 初始分箱：使用等频分箱获得初始分箱
2. 计算每个分箱的违约率
3. 检查单调性：验证违约率是否符合预期趋势（递增/递减）
4. 合并分箱：如果违反单调性，合并违约率差异最大的相邻分箱
5. 重复步骤2-4直到满足单调性或只剩2个分箱

**默认单调性配置：**
- `age`（年龄）: decreasing - 年龄越大，违约率越低
- `income`（收入）: decreasing - 收入越高，违约率越低
- `debt_ratio`（负债比）: increasing - 负债比越高，违约率越高
- `past_due`（逾期次数）: increasing - 逾期越多，违约率越高
- `credit_lines`（信用账户数）: decreasing - 账户越多，信用越好

### 5. 拒绝推论原理

**样本选择偏差问题：**
信用评分模型通常只使用批准客户的表现数据训练，但被拒客户的信用特征往往更差，这导致模型存在样本选择偏差，预测结果偏向乐观。

**三种拒绝推论方法：**

#### 方法1：硬截断法（Hard Cutoff）
1. 使用批准样本训练一个初始模型
2. 用初始模型预测所有被拒客户的违约概率
3. 设置一个截断值（如批准样本违约率的分位数）
4. 概率≥截断值的被拒客户标记为1（违约），<截断值标记为0（正常）
5. 合并批准样本和标记后的被拒样本重新训练

#### 方法2：模糊法（Fuzzy/Parceling）【推荐】
1. 使用批准样本训练一个初始模型
2. 用初始模型预测所有被拒客户的违约概率
3. 按概率从低到高排序被拒客户
4. 前N个标记为0（正常），后M个标记为1（违约），比例与批准样本相似
5. 合并后重新训练模型

#### 方法3：扩充法（Augmentation）【带权重】
1. 使用批准样本训练一个初始模型
2. 每个被拒客户复制两份：一份标记为0，一份标记为1
3. 使用预测概率作为权重：P(0) = 1-p, P(1) = p
4. 使用样本权重训练加权逻辑回归模型

### 7. 评分卡原理

1. **特征分箱（Binning）**：将连续特征离散化（支持单调分箱）
2. **WOE转换**：计算每个分箱的证据权重（Weight of Evidence）
3. **拒绝推论（可选）**：推断被拒客户标签，纠正样本偏差
4. **逻辑回归**：使用WOE值训练逻辑回归模型（支持样本加权）
5. **评分转换**：将逻辑回归输出转换为信用评分

评分计算公式：
```
Score = Offset + Factor * ln(Odds)
其中：
- Factor = PDO / ln(2)
- Offset = Target_Score - Factor * ln(Target_Odds)
```

## 最佳实践建议

1. **数据准备**：
   - 使用真实业务数据替换示例数据
   - 确保数据质量，处理缺失值和异常值

2. **分箱设置**：
   - 根据业务逻辑设置正确的单调性趋势
   - 建议每个特征最终保留3-5个分箱
   - 确保每个分箱有足够的样本量（建议>5%）

3. **拒绝推论**：
   - 被拒率较高（>20%）时强烈建议使用拒绝推论
   - 推荐使用模糊法（Fuzzy），效果稳定且易于解释
   - 可以对比多种方法选择最优方案

4. **模型验证**：
   - 使用包含被拒客户的独立测试集评估模型性能
   - 关注被拒客户群体的预测准确率
   - 定期监控模型表现，适时重新训练

5. **生产部署**：
   - 添加数据验证和异常处理
   - 保存训练好的模型和分箱配置
   - 建立模型监控和更新机制
