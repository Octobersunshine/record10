# Newton-Raphson gRPC Service

这是一个用Go开发的gRPC服务，用于使用**智能牛顿-拉弗森迭代法**求解数学函数的根。

## ✨ 核心特性

### 1. 🔮 自动符号求导

#### 功能说明
服务端**自动计算函数的导函数**，无需用户手动提供导数。基于表达式解析树实现完整的符号微分：

#### 支持的求导法则
- **常数规则**：d/dx(c) = 0
- **变量规则**：d/dx(x) = 1
- **加法规则**：d/dx(u+v) = u' + v'
- **减法规则**：d/dx(u-v) = u' - v'
- **乘积规则**：d/dx(u·v) = u'v + uv'
- **商数规则**：d/dx(u/v) = (u'v - uv') / v²
- **幂函数规则**：d/dx(u^n) = n·u^(n-1)·u'
- **链式法则**：d/dx(ln(u)) = u'/u

#### 自动化简
求导后的表达式会自动化简，例如：
- `d/dx(x^2 + 3x) → 2x + 3`（而非 `2*x^1 + 1*3 + x*0`）
- 去除乘以0、乘以1等冗余项

#### 优势对比
| 方法 | 精度 | 步长依赖 | 计算开销 |
|------|------|---------|---------|
| 数值微分 | 有限精度（舍入误差） | 需选择合适h | 2-3次求值 |
| **符号微分** | **精确** | **无** | 1次解析树遍历 |

---

### 2. 多重根收敛问题修复

#### 问题描述
标准牛顿法在多重根附近只有**线性收敛**，导致收敛极其缓慢。例如求解 `(x-1)^2 = 0` 可能需要上百次迭代才能收敛。

#### 解决方案：自适应修正牛顿法
1. **线性收敛检测**：监测连续两次迭代的步长比，如果 `0.5 < ratio < 1.0` 说明进入线性收敛区域
2. **重数估计**：利用**符号二阶导数**估计根的重数 `m ≈ f'(x)² / (f'(x)² - f(x)f''(x))`
3. **修正迭代公式**：使用 `x_{n+1} = x_n - m * f(x_n) / f'(x_n)` 恢复二次收敛

#### 性能对比

| 函数 | 根的重数 | 标准牛顿法(迭代) | 改进牛顿法(迭代) | 提升 |
|------|---------|-----------------|-----------------|------|
| `(x-1)^2 = 0` | 2 | ~50次 | ~10次 | 5倍 |
| `(x-1)^3 = 0` | 3 | ~100次 | ~15次 | 6-7倍 |

### 3. ✅ 智能输入验证与初始值修正

#### 功能描述
自动检查初始猜测值是否在函数的定义域内：
- 检测奇点（如分母为零的点）
- 检测无穷或NaN求值结果
- 自动提供有效的替代初始值

#### 修正策略
```go
// 检测到初始值不在定义域时，尝试以下备选值
0, 1, -1, 2, -2, 0.5, -0.5, 10, -10
```

#### 示例
```
表达式: 1/x
初始值: 0.0 (奇点)
→ 检测到初始值无效，自动尝试 x=1.0
```

---

### 3. ✅ 复数根自动降级机制

#### 功能描述
当实数迭代发散时，自动切换到复数牛顿法继续求解：

#### 触发条件
1. **实数迭代发散**：`|f(x)| > 1e6 * max(1, |f(x_prev)|)`
2. **导数为零**：`|f'(x)| < 1e-12` 且无法使用二阶修正
3. **迭代达到上限**：实数迭代未收敛

#### 复数迭代策略
- 自动添加微小虚部扰动：`x → x + 0.1i`
- 使用复数算术继续迭代
- 返回复数根结果

#### 示例
```
表达式: x² + 1 = 0
初始值: 1.0 (实数)
→ 检测到发散，自动切换到复数模式
→ 结果: root ≈ 0.0000000000 + 1.0000000000i
```

## Protocol Buffers 定义

```protobuf
service NewtonRaphsonService {
    rpc FindRoot(RootRequest) returns (RootResponse) {}
}

message RootRequest {
    string expression = 1;
    double initial_guess = 2;
    double precision = 3;
    int32 max_iterations = 4;
}

message RootResponse {
    double root = 1;              // 根的实部
    double root_imag = 6;         // 根的虚部
    int32 iterations = 2;
    bool converged = 3;
    string error = 4;
    int32 multiplicity = 5;       // 估计的根的重数
    bool is_complex = 7;           // 是否为复数根
    bool used_fallback = 8;        // 是否使用了降级策略
    string fallback_info = 9;      // 降级信息
    bool input_valid = 10;         // 输入是否有效
    string validation_msg = 11;    // 验证信息
    double suggested_initial = 12; // 建议的初始值
    string derivative = 13;        // 符号求导结果 f'(x)
    string derivative_raw = 14;    // 原始化简前的导数
}
```

## 算法实现细节

### 1. 改进的数值微分
- **一阶导数**（中心差分）：`f'(x) ≈ [f(x+h) - f(x-h)] / (2h)`
- **二阶导数**：`f''(x) ≈ [f(x+h) - 2f(x) + f(x-h)] / h²`
- **复数值导数**：使用复数算术的中心差分

### 2. 重数估计与修正
对于m重根，利用以下关系估计重数：
```
m ≈ f'(x)² / [f'(x)² - f(x)·f''(x)]
```
检测到线性收敛后，应用修正的牛顿公式：
```
x_{n+1} = x_n - m * f(x_n) / f'(x_n)
```

### 3. 发散检测与降级
**实数发散检测**:
```go
math.Abs(fx) > 1e6 * math.Max(1.0, math.Abs(prevFx))
```

**复数切换策略**:
1. 保持当前迭代的实部
2. 添加0.1的虚部作为扰动
3. 使用复数牛顿法继续迭代

## 并发支持

服务端通过以下方式支持并发:
- gRPC默认支持并发处理多个请求
- 使用 `grpc.NumStreamWorkers(10)` 配置工作线程
- 请求计数器使用互斥锁保护
- 每个请求在独立goroutine中处理
